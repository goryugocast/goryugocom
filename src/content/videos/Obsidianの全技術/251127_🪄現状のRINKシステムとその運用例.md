# セミナー構成

### [1. RINKシステムの基本と専用ワークスペース - 知識整理の実行環境](https://knowledgestuck.substack.com/p/rink-7b1?timestamp=1)
**時間**: 00:00:51 ～

Zettlekasten的な知識整理術「RINK」を実践するための専用ワークスペース構築を紹介。「ミュージック専用モード」のような目的別ワークスペースを用意することで、認知負荷を下げながら深い思考を実現。同じファイルを左右に並べるテクニックの活用法を実演し、長大なノート全体を俯瞰しながら、末尾に追加された未整理項目を適切な場所へ移動させるプロセスを解説。このテクニックにより、スクロール中に位置を見失うことを防ぎ、効率的な情報整理が可能になる。

### [2. プレフィックスと無理やり並べる思想 - ボトムアップな知識構築](https://knowledgestuck.substack.com/p/rink-7b1?timestamp=319)
**時間**: 00:05:19 ～

集めた知識の断片に`t.4.a.1`のようなプレフィックスを付けてBaseビューでソート、無理やり並べるプロセスの価値を説明。この「無理やり並べる」という行為が、個々のノート間の関係性を可視化し、視点の変化や知識の欠落を発見する触媒となる。プレフィックスがノートに「順番」という意味を付与し、体系的な知識として機能し始める仕組みを具体例で解説。スケール知識やディミニッシュスケール、フリジアンドミナントスケールなど、音楽関連ノートを例に、ボトムアップな知識構築の実践例を紹介。

### [3. タイトルと内容の継続的改善 - 言語化による体系化](https://knowledgestuck.substack.com/p/rink-7b1?timestamp=481)
**時間**: 00:08:01 ～

ノートのタイトルと内容をガンガン変更し続けることを前提とした運用方法を紹介。単なるテキストではなく、文章として言語化し、章立てとして機能させることの重要性を強調。ディミニッシュスケールの完全形とハーフホール等、複数のバリエーション発見を通じて、知識が有機的に成長していくプロセスを解説。適切なタイトル命名により、ノートが本の見出しのように機能し、より良い理解と検索性を実現する。

### [4. フィルターと絵文字による認知負荷管理 - 視認性と柔軟性](https://knowledgestuck.substack.com/p/rink-7b1?timestamp=784)
**時間**: 00:13:04 ～

Baseプラグインのフィルター機能を活用し、「今見るべき情報」を動的にコントロールする実践法を紹介。`does not start with G`（Gで始まらない）のようなフィルター設定でギター関連ノートを非表示にし、認知負荷を軽減。さらに、カテゴリトップレベルのプレフィックスに絵文字を付与（例：🎹🎸）することで、色による視認性向上とアルファベット順ソート時の上位配置を実現する「ライフハック」を解説。常に全体像を俯瞰できる環境構築の工夫を強調。

### [5. プレフィックスルールの一括変更と柔軟な運用 - プログラムによる効率化](https://knowledgestuck.substack.com/p/rink-7b1?timestamp=1033)
**時間**: 00:17:13 ～

知識整理の構造が大きく変わった際、生成AIを活用してプレフィックスを一括変更できる仕組みを紹介。例えば`T.`で始まるプレフィックスを`TT.`に一括変換するプログラムを生成AIに作成させることで、5分以下の短時間で大規模な変更を実行可能。ルール整備がなされていれば、作業ロスを最小限に抑えながら柔軟な構造変更が可能。Obsidian Publishとの連携状況を把握するため、公開状況をカラムで管理する工夫も紹介。

### [6. 生涯をかけた知的生産システムの育成 - 柔軟性と継続性](https://knowledgestuck.substack.com/p/rink-7b1?timestamp=1401)
**時間**: 00:23:21 ～

RINKシステム最大の思想は「いつでもやり方を変えられる」という柔軟性にあることを強調。ワークスペースは目的に応じて作成・削除、Baseビューのフィルターも自由に変更可能、プレフィックスルールですらプログラムで一括変更できる。知識整理は「整理しながら考える」という負荷の高い行為であり、毎日必死にやる必要はなく、週に2～3回、15分程度の継続で十分。生涯をかけて少しずつ育てる趣味として、自分だけの知的生産システムを構築することの価値を強調。

# メモ・原稿



### 今回の概要

[Obsidianで挫折しないための全技術](https://knowledgestuck.substack.com/p/15obsidian)から続くObsidianシリーズ第9回です。

これまでの全シリーズ一覧はこちら→[🪄Obsidianの全技術 - ごりゅご.com](https://goryugo.com/all-about-obsidian)

今回は、私が実践しているZettelkasten（ツェッテルカステン）的な知識整理術、通称**「RINK」**についてお話しします。このシステムはまだ未完成で実験的な部分も多いのですが、あえてその**“現状”**を皆さんにお見せすることで、ご自身の知識整理のヒントにしていただければと考えています。

### 具体例1：「RINK」を実践するための専用環境 (00:00:51)

まず、特定の目的に集中するため「ミュージック専用モード」のような専用のワークスペースを用意します。その上で、このシステムの核となるのが、一見すると狂気的に見える「同じファイルを左右に並べる」というテクニックです。

これは、一つの長大なノート（**RINK**の中核ノート）を扱う際に絶大な効果を発揮します。左ペインでノート全体の構造（目次）を俯瞰しつつ、右ペインでノートの末尾に追加された未整理の項目を、あるべき場所へと移動させていくのです。これにより、長いノートをスクロールするうちに「今どこを見ているんだっけ？」と迷子になる状態を防ぎます。

### 具体例2：プレフィックスとリンクによるボトムアップな知識構築 (00:05:19)

集めた知識の断片は、まず`t.4.a.1`のようなプレフィックス（接頭辞）を付けてBaseビューでソートし、無理やり並べてしまいます。

一見すると雑なようですが、この「無理やり並べる」という行為が重要です。並べて俯瞰することで、個々のノートの関係性が見えたり、「これも理論の一部だな」といった視点の変化が生まれたり、「この知識が抜けているな」という発見につながったりします。これが、**RINK**におけるボトムアップ的な知識構築のプロセスです。

この手法では、ノートのタイトルや中身は整理しながらどんどん変更していくことを前提としています。プレフィックスを付けたノート群が、**ノートに「順番」という意味を持たせ、体系的な知識として機能し始める**のです。

### 具体例3：フィルターと絵文字で「今見るべき情報」をコントロールする (00:13:04)

情報が増えてくると、すべてを一度に表示するのは非効率です。そこでBaseプラグインのフィルター機能が役立ちます。

例えば、「今はギター関連の知識はいいかな」と思ったら、`does not start with G`（Gで始まらない）というフィルターを設定すれば、ギター関連のノートを一時的に非表示にできます。これにより、認知負荷を下げ、今集中したいことだけに焦点を当てられるのです。

さらに、視認性を高める工夫として「絵文字」の活用があります。カテゴリのトップレベルにあたるノートのプレフィックスに絵文字（例: `🎹` `🎸`）を付けておくと、色が付くことで区切りが明確になります。また、絵文字はアルファベット順でソートした際に上位に来るため、常に全体像を俯瞰できるというライフハックでもあります。

### まとめ：変化し続ける柔軟な知的生産システム「RINK」を育てる

今回ご紹介した**RINK**システムの最も重要な思想は、「いつでもやり方を変えられる」という柔軟性です。

*   **ワークスペース**は、目的に応じて作ったり消したりできる。
*   **Baseのビュー**も、興味や進捗に合わせてフィルターをかけたり、カラムを増減させたりできる。
*   **プレフィックスのルール**ですら、プログラムを使えば一括変更が可能です。

知識の整理は「整理しながら考える」行為であり、非常に負荷が高いものです。毎日必死にやる必要はありません。週に数回、15分でも向き合う時間があれば十分です。生涯をかけて少しずつ育てていく趣味として、自分だけの知的生産システムを構築してみてはいかがでしょうか。
