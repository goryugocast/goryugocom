# ノートリファクター機能で永続化したいメモを切り出す

## 概要
Obsidianのノートリファクター機能を使って、デイリーノートに書いた断片的なメモを新しい独立したノートに切り出し、永続的な知識に変換するプロセス。文脈を失わずに、整理と育成が可能になる。

## ノートリファクター機能とは

### 基本機能
- デイリーノートの一部を選択
- 新しいノートとして切り出す
- 元のノートには切り出したノートへのリンクが自動的に残る

### プラグイン情報
- 標準のObsidian プラグイン（Note Refactor）
- コマンド：「Cmd + Shift + N」（カスタマイズ可能）

## 実装フロー

### 1. メモを選択
```
- ハーフディミニッシュの理論と使用例
- マイナー7b5コードの機能性
- 実践的な運指パターン
```

### 2. ノートリファクターを実行
1. 選択範囲を自動的に新しいノートに変換
2. 自動的にノートタイトルを生成
3. 元のデイリーノートには `[[新しいノート]]` というリンクが残る

### 3. 新しいノートで詳細化
- 切り出したメモを整理
- 関連情報を追加
- タグやリンクを付ける

## メリット

### 文脈の保全
- 「このメモはいつ作成されたか」がデイリーノートのリンクから明白
- バックリンク機能で「この日に何を考えていたか」が追跡できる
- メモが断片的でも、作成日のコンテキストで補完される

### ノートの成長
- 小さなメモから始まる
- 週の振り返りで関連情報を追加
- 段階的に豊かなノートに成長していく

### デイリーノートのコンパクト化
- 詳細なメモはノートリファクターで別ファイルに移す
- デイリーノート自体は短く保つ
- ただしリンクが残るため、後で追跡可能

## 実例：音楽理論メモ

### デイリーノートのメモ
```
ハーフディミニッシュのこと調べた
- バーテノール系とか
- 使い方によってマイナーコードみたいに聞こえる
```

### ノートリファクター実行後
```
[[ハーフディミニッシュコードの理論と実践]] にリンク変更
```

### 新規ノートの内容
```
- ハーフディミニッシュ (m7b5) の理論
- II-V進行での機能
- 実践的な運指パターン
- 関連ノートへのリンク
```

### 後の改善
- 他のメモに「ハーフディミニッシュ」の言及があれば、自動的にリンクできる
- バックリンク表示で「複数の日付で言及されている」ことが見える
- 関心の深さを認識できる

## 永続化のプロセス

### 週次レビュー時に実行
1. 先週のデイリーノートを振り返る
2. 「後で発展させたい」というメモを選別
3. ノートリファクターで切り出す
4. 後の振り返りで情報を追加

### タグとの組み合わせ
- ノートリファクター後、`#review` タグを付ける
- Spaced Repetitionで定期的に見返す
- 段階的に知識が定着していく

## 注意点

### 完璧さの不要性
- ノートリファクターで作成したノートは「完成形」ではない
- むしろ「成長途中のノート」と考える
-後で何度も編集・改善される前提

### タイミング
- 毎日実行する必要はない
- 週の終わりにまとめて実行
- あるいは「このメモは重要」と判断した時だけ

### ファイル名の決定
- ノートリファクターは自動的にタイトルを生成するが、編集可能
- 後で検索しやすい名前に変更すると便利
