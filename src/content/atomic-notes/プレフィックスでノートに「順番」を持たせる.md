# プレフィックスでノートに「順番」を持たせる

## 基本概念

プレフィックスの真の価値は、**ノートに「順番」という意味を持たせる**ことで、単なるリストを**体系的な知識へと変える**ことにある。

## プレフィックス構造の例

`t.4.a.1` というプレフィックスの場合：

| 要素 | 意味 | 例 |
|-----|------|-----|
| t | カテゴリトップレベル | t = Theory（理論） |
| 4 | グループ番号 | 4 = 4番目のグループ |
| a | 小グループ | a = 最初のサブカテゴリ |
| 1 | アイテム番号 | 1 = 1番目のアイテム |

## 「順番」が生む効果

### Baseビューでソートしたときの動作

1. プレフィックス順にソート
2. 関連するノートが自動的に近い位置に集まる
3. 「この項目の次は、自然とこの項目が来る」という順序が形成される

### 例：理論カテゴリの場合

```
t.4.a.1 ディミニッシュスケール
t.4.a.2 ハーフホール
t.4.a.3 オルタードスケール
t.4.b.1 トニックディミニッシュ
t.5.a.1 チャーチモード
...
```

## 従来のファイル名との違い

### ファイル名でのソート
- 長さが不均一
- アルファベット順に縛られ、意図的な順序付けが難しい
- 「理論の4番目グループの最初」という意図が不明確

### プレフィックスでのソート
- 統一的な構造
- **意図的に「順番」を付与**できる
- 本の目次のような階層構造が実現

## プレフィックスによる体系化の強み

- **ファイル名は柔軟に変更可能**（プレフィックスは固定）
- **プレフィックスで体系が決まり、ファイル名は内容に最適化**できる
- 例：プレフィックス `t.4.a.1` のままで、「ディミニッシュスケール完全形」に改題可能

## 運用のコツ

- プレフィックスを付けて「とりあえず並べる」
- Baseビューで眺めて、関係性を確認
- 必要に応じてプレフィックスを調整
- （大規模変更は生成AIで一括処理可能）
