# ファイルリカバリーは必ず有効化する

## 概要

Obsidianのコアプラグインである「File Recovery」（ファイルリカバリー）は、必ず有効化しておくべき機能。ノートの編集履歴を自動的に保存し、誤って削除したり上書きしたりした場合でも復元できる。特にスマホでの誤操作や同期トラブルの際に、この機能があれば安心。データを失うリスクを大幅に減らせる重要な保険。

## 重要なポイント

**ファイルリカバリーの機能**:
- ノートの編集履歴を自動保存
- 削除したファイルの復元
- 誤って上書きした内容の復元
- 同期トラブル時のバックアップ

**なぜ必須なのか**:
- スマホでの誤操作が起こりやすい
- 同期の不具合でデータが消える可能性
- 復元できないと大きな損失
- 保険として必ず入れておく

**設定場所**:
- `設定 > Core plugins > File recovery`を有効化
- 特に細かい設定は不要
- デフォルト設定で十分機能する

## 実践のコツ

- Obsidianを使い始めたら最初に有効化
- 定期的にバックアップも取る（追加の保険）
- 復元が必要になったら、設定から復元メニューにアクセス
