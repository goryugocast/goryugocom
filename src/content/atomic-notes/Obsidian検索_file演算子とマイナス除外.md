# Obsidian検索：file演算子とマイナス除外

`file:`演算子でファイル名を限定し、`-（マイナス）`で検索結果から除外することで、目的の情報をピンポイントで抽出できます。これらは実装が簡単で、効果が大きい基本演算子です。

## file演算子の使い方

```
file:日本語の音
```
ファイル名に「日本語の音」というキーワードを含むノートだけを検索対象にします。

## マイナス除外演算子の活用

```
ドミナント -path:daily/
```
「ドミナント」を含むノートから、デイリーノートを除外します。

```
-file:weekly
```
ウィークリーノートを検索結果から除外し、整理済みノートだけを表示。

## 実践的な活用シーン

**シーン1**: 整理状況の把握
- `path:daily/ ドミナント` で検索して、デイリーに未整理のドミナント関連情報を発掘
- `-file:weekly` を追加して、ウィークリーサマリーを除外し、本体ノートだけを表示

**シーン2**: 特定フォーマットのみ検索
- `-file:W` で、ウィークリーノート（日付がWで始まる）を除外

## 組み合わせのポイント

`path:` と `-` の組み合わせだけで、大半の検索ニーズを満たせます。まずはこの2つの演算子を使いこなすことを最優先にしましょう。
