# デイリーノート管理とフォルダ構造

Obsidianの検索機能を効果的に活用するには、デイリーノートと他のノートを物理的に分けるフォルダ構造が重要です。これにより、`path:` 演算子による検索精度が大幅に向上します。

## 推奨フォルダ構造

```
/Daily
  - 2025-10-01.md
  - 2025-10-02.md
  - 2025-10-03.md

/Topics
  - ドミナントについて.md
  - 遺伝子について.md
  - Ableton_Pushについて.md

/Projects
  - プロジェクトA/
  - プロジェクトB/

/Books
  - 読了記録/
  - 書籍引用/
```

## デイリーノートを別扱いする理由

### 整理不要の容器としての機能
- 思いついたことをそのまま記録
- タグやフロントマターなしで記述
- 分類や体系化の負担なし

### 検索での除外が効果的
```
-path:daily/ ドミナント
```
デイリーノートを除外することで、整理済みノートだけを対象にした検索が可能。

### 段階的な整理の基点
デイリーに書いたことから、後で「これは整理ノート化すべき」な領域を発掘します。

## 他のノートとの分け方

**デイリーノート**: 日付形式（2025-10-02）、分類不要

**整理済みノート**: テーマごとに分類されたフォルダに配置

この分け方により、検索時に：
- `path:daily/` で整理前の情報源を検索
- `-path:daily/` で整理済みノートだけを検索

という相互補完的な検索が実現します。

## デイリーノートの内容について

デイリーノートには以下の全てが入ります：
- 思いついたアイデア
- 会議記録
- 読書メモ
- 学習内容
- 時間記録

整理が追いついていない領域に、後から `path:daily/` で検索でアクセスする仕組みです。
