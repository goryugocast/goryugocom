# Move file to another folderコマンドの活用

## 概要

Obsidianの「Move file to another folder」コマンドは、現在開いているファイルを別のフォルダに移動させる機能。読書メモを読み終えた時に、作業中フォルダからBooksフォルダに移動させるなど、ノート管理のワークフローで頻繁に使う場合は、ショートカットキーを割り当てておくと便利。

## 基本機能

### ファイルの移動
- 現在開いているファイルを、任意のフォルダに移動
- マウス操作（ドラッグ&ドロップ）より速い
- リンクは自動的に更新される

### フォルダの選択
- コマンド実行後、移動先フォルダを選択する画面が表示される
- インクリメンタルサーチで素早く選択できる

## 活用シーン

### 読書メモの整理
1. 本を読み終えたら、このコマンドを実行
2. `books/2022/` などのフォルダを選択
3. 読書メモが整理される
4. 「現在読んでいる本」のリストから自動的に消える

### ノートのライフサイクル管理
- ドラフト → 完成 → アーカイブのような移動
- プロジェクトフォルダ → 完了フォルダへの移動
- 一時的なメモ → 永続的なノートへの昇格

## ショートカットキーの設定

### 設定方法
1. 設定 > ホットキー を開く
2. 「Move file to another folder」を検索
3. 任意のショートカットキーを割り当てる（例：`Cmd+Shift+M`）

### 推奨設定
- よく使う場合は、覚えやすいショートカットを設定
- 他のコマンドと競合しないように注意

## 実践のコツ

### ワークフローに組み込む
- 読書が終わったら、タグ付け→フォルダ移動の流れを習慣化
- 定期的な整理の時間を設けて、まとめて移動してもよい

### リンク切れの心配は不要
- Obsidianがリンクを自動的に更新してくれる
- 他のノートからのリンクは維持される

### フォルダ構造をシンプルに保つ
- 移動先のフォルダが多すぎると、選択が面倒
- 必要最小限のフォルダ構造にしておく
