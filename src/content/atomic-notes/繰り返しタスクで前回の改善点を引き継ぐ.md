# 繰り返しタスクで前回の改善点を引き継ぐ

## 概要
セミナー配信、記事投稿など繰り返しタスクを実行する際、前回のタスクノートに記載された改善点や注意事項を、次回のタスクに自動的に引き継ぐシステム。Duplicate機能で実現される。

## 改善の螺旋的向上

### 従来の方法（テンプレート）の問題
```
テンプレート使用の場合：
1回目 → 実行して気づき「次は Xなやり方を試す」と記録
2回目 → テンプレートから新規作成（気づきが反映されない）
3回目 → また同じ試行錯誤
```

### Duplicate活用の場合
```
1回目 → 実行して気づき「次は Xなやり方を試す」と記録
2回目 → 前回ファイルを Duplicate（改善点が自動的に含まれる）
3回目 → さらに改善を加えて Duplicate
```

## 実装フロー

### タスク実行後の記録
```markdown
## 250826 セミナーアーカイブ配信

### 実行結果
- 動画編集完了
- 字幕挿入：1時間30分かかった

### 改善点（次回用）
- Final Cut Pro での編集は時間がかかりすぎ
- 次回は DaVinci Resolve を試す
- 現像プリセットを保存しておく
- BGM の挿入位置を決めておくと効率化される
```

### Duplicate実行時
1. 前回のファイルをコマンドパレットで「Duplicate current file」
2. ファイル名を新しい日付に変更（例：250902）
3. 自動的に「次回は DaVinci を試す」という改善点がコピーされる

### 改善の実行
```markdown
## 250902 セミナーアーカイブ配信

### 実行内容
- 動画編集：DaVinci Resolve を使用（前回の改善点を実施）
- 編集時間：45分（前回比 45分削減）

### さらなる改善点（次回用）
- DaVinci のプリセットを3種類作成した
- BGM ライブラリを事前に整理した
- 次回は「色補正」の工程を加える
```

## メリット

### 継続的なプロセス改善
- 毎回のタスクが、前回の経験を含む
- 同じ仕事でも、1ヶ月後には改善されている
- 「やり方がどんどん効率化される」という実感が得られる

### 知識・スキルの蓄積
- 単に繰り返すのではなく、進化しながら繰り返す
- 過去のファイルを遡ると「このタスクの成長過程」が見える

### 時間短縮
- 改善の試行錯誤時間が減る
- すでに検証済みの方法が組み込まれている

## 実例：セミナー配信タスク

### 初回（250819）
```
- 動画編集に 2 時間かかった
- Final Cut Pro の操作に時間がかかった
- 次回は別のツールを試す
```

### 2回目（250826）- DaVinci を試した
```
- 動画編集に 90 分かかった
- DaVinci は早い
- ただし、プリセット設定に時間がかかった
- プリセットを保存しておく
```

### 3回目（250902）- 効率化実施
```
- 動画編集に 45 分で完了
- プリセット活用で時間短縮
- 次は色補正を組み込む
```

### 4回目（250909）- さらに進化
```
- 色補正を組み込んで 50 分で完了
- クオリティが向上した
- 次は AI 補正を試してみる
```

## 改善点の記述方法

### セクション化
- 「実行内容」：実際に何をしたか
- 「所要時間」：どれだけかかったか
- 「改善点」：次に試すこと

### 具体性の重要性
```
× 「編集をもっと早くする」
○ 「DaVinci Resolve を試す。プリセットを3つ作成。色補正の時間計測。」
```

### 数値化
- 所要時間：実際の時間を記録
- 「前回比 30% 削減」などの改善幅を見える化

## 継続のコツ

### 完全さの不要性
- 全ての改善を実施する必要はない
- 「今回は 1 つだけ試す」でいい
- 段階的な改善で十分

### 失敗の記録も重要
```
× 「XX を試したが、失敗した」← 記録しない
○ 「XX を試したが、YY のせいで失敗。次は ZZ にする」← 記録する
```

失敗の原因を記録しておくことで、次回の同じ失敗を回避できる。

## DataViewとの連携

### シリーズ全体の改善トレンド
- DataView で過去 10 回のセミナー配信ノートを一覧
- 「どのタイミングで、どの改善が入ったか」が視覚的に分かる
- プロセス進化の歴史が一目瞭然
