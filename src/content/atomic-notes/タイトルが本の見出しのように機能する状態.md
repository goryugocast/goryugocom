# タイトルが本の見出しのように機能する状態

## 理想的な状態

RINKにおける最終的な目標の1つが、**各ノートのタイトルが本の「見出し」のように機能する**こと。

つまり：
- タイトルを見ただけで、内容がおおよそ推測できる
- 階層的な見出し体系が自然に形成される
- ネットワークの中で、その知識の位置づけが明確

## 本の見出しとの対応

### 従来の本の構成
```
第1部 基礎知識
  第1章 スケールとは
    1.1 スケールの定義
    1.2 メジャースケール
    1.3 マイナースケール
  第2章 発展的なスケール
    2.1 ペンタトニック
    2.2 ディミニッシュスケール
...
```

### RINK（理想的）での構成
```
🎵 Music Theory
├─ t.1.a.1 スケールとは
├─ t.1.a.2 メジャースケール
├─ t.1.a.3 マイナースケール
├─ t.2.a.1 ペンタトニック
├─ t.2.a.2 ディミニッシュスケール
│  ├─ t.2.a.2.1 完全形の構造
│  ├─ t.2.a.2.2 ハーフホール
│  └─ t.2.a.2.3 実践活用
```

Baseビューで見ると、プレフィックスによる階層構造が見出しのように機能。

## これが実現するメリット

### 1. 瞬間的な理解
- タイトルだけで内容が推測できる
- 「あ、あの知識だ」と即座に判断可能

### 2. 検索効率の向上
- キーワード検索時に、適切なノートが見つかりやすい
- 関連ノートとのリンク構築が容易

### 3. 学習者視点での価値
- 初心者が見た時、本の目次のように進められる
- ノート群全体が1つの「教科書」として機能

### 4. 知識構造の可視化
- ネットワークビューで見ると、知識の体系が明確
- 「この知識の周辺には、こんな知識があるんだ」と発見できる

## タイトル設計のコツ

### 具体性を重視
❌ 悪い例：「スケール」「理論」
⭕ 良い例：「ディミニッシュスケール完全形の構造と応用」

### 階層構造を反映
プレフィックスの深さに応じて、タイトルの詳細度も調整
- 浅い階層：大項目的なタイトル
- 深い階層：具体的で限定的なタイトル

### 検索性を考慮
- 一般的すぎないこと
- キーワードを含むこと
- 同じ概念に複数の呼び方があれば、主要なものを優先

## 実現までのプロセス

### 初期段階
最初は、タイトルが曖昧でも構わない。重要なのは「記録される」こと。

### 学習進行段階
学習が進むにつれて、より具体的で適切なタイトルに改題。

### 成熟段階
充分な知識が蓄積されると、自然と「本のような見出し体系」が形成される。

この段階に到達したとき、RINKは単なる「ノート管理ツール」から、**知的生産の中核インフラ**へと進化する。
