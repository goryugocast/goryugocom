# 接頭辞・ドット区切りでAI・ターミナル対応命名

## 概要
`ks.rem.040`のような接頭辞とドット区切りを使用することで、AIツールやターミナルコマンドとの相性が劇的に向上する。

## ドット区切りの利点

### AI時代への対応
生成AIにファイル処理を指示する際、ドット区切りは非常に扱いやすい。
```
"ks.rem.040というプレフィックスのファイルを処理して"
```
という指示が直感的に伝わる。

### ターミナル互換性
```bash
# プレフィックスに基づくファイル処理
ls ks.rem.*  # 同じシリーズのファイルをすべて表示
grep -r "ks.rem" .  # プレフィックスで検索
```
シェルスクリプトとの親和性が高い。

## ファイル名の構造化例

### 3段階構成
```
プロジェクト.カテゴリ.連番
ks.rem.040
```

### より詳細な構成
```
プロジェクト.サブプロジェクト.カテゴリ.連番
ks.newsletter.seg1.001
```

## クイックスイッチャーでの活用

Cmd+Oで検索する際、プレフィックスの一部を入力するだけで該当ファイルがフィルタリング表示される。
```
"ks.rem"で検索 → セミナーリマインド関連のファイルのみ表示
"ks.seg1"で検索 → セグメント1関連のファイルのみ表示
```

## 実装の注意点

- 数字を3桁（001, 002...）または2桁（01, 02...）に統一
- プロジェクト・カテゴリは最大3階層まで
- 英数字のみ使用（AIやターミナルの互換性確保）
- 連番の桁数を統一することでソート順序が意図通りになる
