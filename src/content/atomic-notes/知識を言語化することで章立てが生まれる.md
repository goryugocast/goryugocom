# 知識を言語化することで章立てが生まれる

## 基本概念

RINKで最も重要な活動の1つが、**知識を言語化する**こと。単なるテキスト記録ではなく、「**文章として読めるレベル**」に整理することで、知識が初めて「体系的」な形をなす。

## 言語化の3段階

### 段階1：断片的な記録
```
- ディミニッシュスケール
- 2セント下げたすべての音
- 完全形とハーフホール
```
→ これだけではメモレベル

### 段階2：簡潔な説明
```
ディミニッシュスケール：
半音と全音が交互に並ぶスケール。
完全形（半→全→半...）とハーフホール（全→半→全...）の2種類がある。
```
→ 情報として成立

### 段階3：体系的な文章
```
ディミニッシュスケール完全形とハーフホール

ディミニッシュスケールは、2つのバリエーションが存在する。

完全形：半音と全音が規則正しく交互に並ぶ。
C = C, Db, Eb, E, F#, G, A, B

ハーフホール：異なるパターン。
C = C, D, Eb, F, F#, G#, A, B

用途の違い：
- 完全形：速いパッセージに
- ハーフホール：特定の和音との相性

関連スケール：[[オルタードスケール]]、[[チャーチモード]]
```

→ これが本の「章立て」

## 言語化のメリット

### 1. 自分の理解度が可視化
- 「文章で説明できない知識は、まだ理解していない」
- 言語化する過程で、知識の曖昧さが露わになる

### 2. 他者への説明が容易
- 言語化された文章は、他の人にも理解しやすい
- 知識の共有が可能に

### 3. 後からの検索・活用
- テキストとして記録された知識は、検索性が高い
- 他のノートからリンクしやすい

### 4. 体系的な構造が見える
- 文章の段落分けそのものが、知識の構造を示す
- 見出し、本文、まとめの構成が自然に生まれる

## RINK運用での実装

### スケール知識の成長例

#### 初期
```
スケール: 複数の音の集合
```

#### 学習進行中
```
スケール: 複数の音が規則に基づいて配置されたもの。
メジャー、マイナー、ペンタトニック、ディミニッシュなど多数。
```

#### 深い理解段階
```
スケールの本質と機能

スケールとは、特定の音程関係に基づいた音の集合。
トライアド（3音）の組み合わせによって生まれる色彩が、スケール選択の基本となる。

例えば、ディミニッシュスケールは：
- トニックディミニッシュコード上での使用
- 速いパッセージの構築
- 半音階的なテンション作成
...
```

この段階で、知識が「文章」→「体系」へと進化する。

## 重要な注意点

言語化は、単なる「きれいに書く」ではなく、**自分の理解を正確に文章化する作業**。この過程で初めて、断片的な知識が体系的な知識へと変換される。
