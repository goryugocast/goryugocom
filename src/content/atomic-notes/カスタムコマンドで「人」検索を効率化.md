# カスタムコマンドで「人」検索を効率化

## 概要
Another Quick Switcherのカスタムコマンド機能を使い、特定の人物に関連するすべてのノートを素早く検索するコマンドを作成する。

## 実装例

### 人物検索用カスタムコマンド

#### 設定内容
```
コマンド名: People Search
プリフィックス: @
検索対象: people プロパティ
```

#### 使用例
```
@うりなみ
→ people: うりなみ というプロパティを持つノートをすべて表示
```

結果表示：
- レッスン記録
- 打ち合わせノート
- ディスコード上のやり取りメモ
- 提案資料
- その他該当ノート

すべてが一度に表示される。

## メリット

### 1. 人物ベースの情報統合
同じ人物に関する複数の種類のノートを、一度に参照できる。

### 2. 学習記録の一元化
特定の生徒さんのレッスン記録、学習進捗、質問記録などが自動集約される。

### 3. クライアント管理
クライアントごとのすべての資料、打ち合わせ記録、納品物などを素早く把握。

### 4. コラボレーション効率化
チームメンバーとのやり取り、共有資料、進捗記録がひとまとめ。

## 実装のポイント

### people プロパティの統一

```yaml
# 良い例（統一された命名）
people: うりなみ
people: 山田太郎
people: 相川さん

# 悪い例（表記ゆれ）
people: うりなみ
people: ウリナミ
people: 売値波
```

表記ゆれがあると、カスタムコマンドでの検索漏れが発生。

### 複数人の場合

```yaml
people: うりなみ, 山田太郎
# または
people:
  - うりなみ
  - 山田太郎
```

複数人とのやり取りの場合、カンマ区切りまたは配列形式で記入。

## 活用シーン

### シーン1：レッスン講師の場合
```
@山田花子
→ 山田さんのレッスン記録、進捗メモ、宿題確認がすべて表示
```

### シーン2：営業・契約業務
```
@A社
→ A社との打ち合わせ、提案資料、契約書、請求書がすべて表示
```

### シーン3：Podcast出演者
```
@相川さん
→ 相川さんとの録音日時、台本、実際の会話記録、配信後のメモ
```

## 他のコマンドとの組み合わせ

### 複合検索例
```
通常のCmd+O: ファイル名 + people プロパティ
@people: people プロパティのみ（専門検索）
#project: プロジェクト検索（追加で実装）
```

3つのコマンドで、ほぼすべての検索ニーズをカバー。

## 注意点

- 人物名の綴りは最初に決めて、ローカルルール化する
- 初期段階では既存ノートのpeople プロパティを一括確認・統一
- 新規ノート作成時は、ローカルルールに従う癖をつける
