# Baseプラグインのthis.queriesで動的フィルタリング

## 概要
`this.people`のような動的クエリを使用することで、現在開いているノートのプロパティに基づいて関連ノートを自動フィルタリング可能。

## 仕組み

### 基本的な構文
```
people: this.people
```
現在のノートが`people: 相手名`というプロパティを持つ場合、同じ`people`値を持つすべてのノートを表示。

## 実用例

### 営業・打ち合わせ管理
同じ取引先との複数のやり取り（メール、会議記録、提案資料など）を一度に参照可能。

### プロジェクト管理
同じプロジェクトに関連するすべてのファイル（告知、進行状況、アーカイブなど）を自動集約。

### 人物関連の情報収集
レッスン、ディスコード上のやり取り、文字起こしなど、同じ人物に関するあらゆる記録を一覧表示。

## メリット

1. **手動の絞り込みが不要**
2. **新しいノートが追加されると自動で表示される**
3. **プロパティ値の変更で即座に反映**

## 応用例
- `this.topic` - 同じトピックの関連ノートすべて
- `this.project` - 同じプロジェクトの関連ファイルすべて
