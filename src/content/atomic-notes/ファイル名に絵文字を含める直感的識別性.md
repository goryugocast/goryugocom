# ファイル名に絵文字を含める直感的識別性

## 概要
ファイル名に絵文字を含めることで、ファイルの内容を瞬間的に識別でき、検索と管理の効率が向上する。特に数が増えたVault環境で効果的。

## 絵文字の役割

### 視覚的アイコン化
```
🔔 リマインド（通知）
📢 告知・告知ページ
📦 アーカイブ配信
⚙️ 準備・セットアップ
🎙️ ポッドキャスト関連
```
ファイル一覧を見ただけで、ファイルの性質がすぐわかる。

## 絵文字使用のメリット

### 1. 検索スピードの向上
```
🔔で始まるファイルを探したい
→ 一目で視認できる
```

### 2. クイックスイッチャーでの視認性向上
Cmd+Oで表示されたファイル一覧が、絵文字により素早くスキャン可能。

### 3. 日本語とのバランス
日本語テキストだけで区別するより、視覚的記号（絵文字）の方が瞬間的。

### 4. 国際対応
言語に依存しない視覚記号として機能。

## AI時代の注意点

### 課題：AIへのファイル指定が困難
```
# 日本語は簡単
"セミナー直前リマインドのファイルを処理して"

# 絵文字は厄介
"🔔がついてるファイルを処理して" → 指示が不明確
```

## 解決策

### ハイブリッド命名規則
```
ks.rem.040_🔔セミナー直前リマインド.md
```
- 人間向け：ファイル名全体で視認性確保
- AI向け：プレフィックス（ks.rem）で識別
- 絵文字はあくまで補助的役割

### AIへの指示方法
```
"ks.rem というプレフィックスのファイルを処理して"
# → AIはプレフィックスで識別、絵文字は無視
```

## 推奨される絵文字の選定基準

- **シンプル**: 複雑な絵文字は避ける
- **一貫性**: 同じ意味なら同じ絵文字を使う
- **限定数**: 5～10種類程度に絞る
- **ローカルルール化**: チーム・個人で統一
