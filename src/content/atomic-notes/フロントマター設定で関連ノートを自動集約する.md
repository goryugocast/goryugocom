# フロントマター設定で関連ノートを自動集約する

## 概要
打ち合わせノートなどに`context`や`people`といったプロパティを設定しておくだけで、Baseプラグインが関連ノートを自動で集約・表示するメカニズム。

## 具体的な活用例

### 例1：最近の打ち合わせ一覧
```
context: meeting
```
このプロパティを持つノートをすべて抽出し、新しい順に5個だけ表示するようにBaseで設定可能。

### 例2：特定の人との打ち合わせ履歴
```
people: うりなみ
```
このプロパティを持つすべてのノートを表示。前回の打ち合わせ内容をすぐに参照できる。

### 例3：動的クエリで現在ノートに基づいた関連ノート表示
```
people: this.people
```
現在開いているノートの`people`プロパティと同じ値を持つノートだけを自動表示。

## 実装のメリット

1. **手動検索が不要** - 毎回手動で探す手間が削減される
2. **コンテクスト維持** - 同じプロジェクト・人物に関する情報が一堂に
3. **複製での継承** - ファイルを複製するたびに設定が引き継がれる

## 注意点
- プロパティは統一した値を使う（表記ゆれに注意）
- 多すぎるプロパティは管理が煩雑になる
