# Obsidian検索：path演算子でフォルダ検索

`path:`演算子は検索対象を特定のフォルダに限定する最も実用的な検索演算子です。ノートの整理が追いついていない初期段階で特に強力な機能です。

## 基本的な使い方

```
path:daily/
```
このように記述するとデイリーノートフォルダの中だけを検索対象にします。

## 実践例

- `path:daily/ ドミナント` → デイリーノート内から「ドミナント」に関する記述を検索
- `path:projects/2025Q1/` → 特定プロジェクトフォルダに限定した検索
- `path:music/ スケール` → 音楽関連ノートからスケール関連の情報を抽出

## 組み合わせのコツ

検索精度を高めるため、他の演算子と組み合わせることが重要です。特にデイリーノートは「何でも入れてOK」という位置づけなので、`path:daily/`で限定することで、本体ノートや整理済みノートとの重複を避けられます。

## わかりやすい活用シーン

新しいノートを作成した後、「このトピックについての整理ノートは作ったのに、デイリーに散らばった情報はあるだろうか？」という問い合わせに最適です。
