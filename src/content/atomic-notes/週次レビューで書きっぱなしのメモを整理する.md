# 週次レビューで書きっぱなしのメモを整理する

## 概要
デイリーノートに書き込まれた断片的で雑なメモを、週ごとにまとめて整理・永続化するプロセス。知識管理システムの中で、最も重要な仕組みの一つ。

## 週次レビューの位置づけ

### デイリーノート管理の段階
```
デイリーノート作成
  ↓
（毎日）メモの蓄積（断片的・未整理）
  ↓
週次レビュー（ここで整理）
  ↓
永続的なノートへの進化
```

### レビューの頻度
- 週の終わり（金曜日か日曜日）に実施
- または月曜朝に先週を振り返る
- 最低でも週 1 回は必須

## 実装プロセス

### ステップ 1: 先週のデイリーノートを開く
```
Monday（08/21）～ Sunday（08/27）の 7 つのノートを順番に見る
```

### ステップ 2: 各ノートをスキャン
- 重要なメモを発見
- 「後で発展させたい」と判断したメモに目印（例：★）を付ける
- あるいは、重要なメモを選別

### ステップ 3: ノートリファクターで切り出す
1. 重要なメモを範囲選択
2. コマンド + Shift + N（ノートリファクター）実行
3. 新しいノートが自動生成
4. デイリーノートには元のノートへのリンクが残る

### ステップ 4: 新規ノートを整理・発展させる
```
デイリーノート：
「ハーフディミニッシュの理論について調べた」
  ↓ ノートリファクター
[[ハーフディミニッシュの理論と実践的運指]]
  ↓ 整理・詳細化
新規ノートに：
- 定義
- 実際の使用例
- 関連ノートへのリンク
- #review タグを付与
```

## メリット

### ノートの成長メカニズム
- デイリーノートの粗いメモから始まる
- 週単位で育成される
- 数週間後には充実した知識ノートに進化

### デイリーノートのコンパクト化
- 詳細な内容は別ノートに移動
- 元のデイリーノートはリンクのみになり、短く保たれる
- 後の振り返りも容易

### 情報の体系化
- バラバラだったメモが、カテゴリごとにまとまる
- リンクで結ぶことで、知識ネットワークが構築される

## 実例

### 先週のデイリーノート（乱雑な状態）
```
## 08/21（月）
- 打ち合わせ終了
- チャットGPTでハーフディミニッシュの使い方聞いた
- こんな理論があるんだ

## 08/22（火）
- ハーフディミニッシュとm7b5の関係性について考察中
- 実際の曲での使用例を探してみた

## 08/23（水）
- ハーフディミニッシュはII-V進行で使える
- マイナーキーでのコード進行の重要性を認識

...etc
```

### 週次レビュー後
```
# デイリーノート（整理済み）
- 08/21: [[ハーフディミニッシュの理論と実践的運指]]
- 08/22: 営業会議
- 08/23: 新規企画の検討

# 新規ノート
[[ハーフディミニッシュの理論と実践的運指]]
- 定義：m7b5（マイナー7フラット5）コード
- II-V進行での機能性
- 実際の運指パターン
- 関連キー：[[コード理論]]、[[ジャズハーモニー]]
```

## 作業時間

### 想定時間
- 1週間分のレビュー：20 ～ 30 分
- ノートリファクター：5 ～ 10 分 / 項目
- 新規ノートの整理：15 ～ 30 分 / 項目

### 効率化のコツ
- 全て完璧に整理する必要はない
- 「整理できそうなメモ」だけ対象にする
- 複雑そうなメモは、翌週に回してもいい

## 継続のポイント

### 形式の柔軟性
- 毎週同じ時間にする必要はない
- あるいは、月に 1 回の大規模レビューでもいい
- ただし「何らかのペース」で継続することが重要

### 完璧さへの執着の排除
- 「全てのメモを整理する」のは目標ではない
- 「重要そうなものだけ」に限定
- その方が継続性が高まる

## 注意点

### 後追い作業の回避
- レビュー時点で、ノートは「概略版」でいい
- 詳細化は後で、必要に応じて実施
- 完璧版を目指すと、レビュー自体がストレスになる

### タグの活用
- 週次レビュー後は `#review` タグを付与
- Spaced Repetition で定期的に見返す
