# 知識整理は週に2～3回、15分程度が継続的

## 基本原則

RINKの運用において、最も過小評価されている重要な原則が、**「毎日必死にやる必要はない」**という指摘。むしろ、**週に2～3回、15分程度の継続が理想的**。

## なぜこの頻度なのか

### 知識整理の負荷の高さ

知識整理は「**整理しながら考える**」という、非常に**認知負荷の高い行為**。

```
通常の作業：         既知のタスクを処理
知識整理：          同時に思考し、分類し、関係性を考え、記録する
                  → 脳への負荷が桁違い
```

### 毎日やると起こる問題
- 燃え尽き症候群（バーンアウト）
- 集中力の低下
- 整理作業が雑になる
- 結果的に続かない

### 週2～3回の継続が現実的
- 心身への負荷が最適化
- 集中力が維持される
- 「趣味」として楽しめるレベルが保持される

## 現実的な運用スケジュール

### 週のパターン例

| 曜日 | 活動 | 時間 |
|-----|------|------|
| 月 | 日常の記録 | 5分（Discordへのメモなど） |
| 火 | - | - |
| 水 | 知識整理セッション | 15分（Baseビューでの配置調整） |
| 木 | 日常の記録 | 5分 |
| 金 | - | - |
| 土 | 知識整理セッション | 20分（タイトル改題、リンク追加） |
| 日 | 週次レビュー | 10分（振り返り） |

→ 週計：約55分

この程度で、十分に知識が成長し、体系化が進む。

##「15分」でできる作業

### 実践的な例

#### 1. プレフィックス配置の最適化（15分）
- Baseビューで未配置のノートを確認
- 最適なカテゴリを判断し、プレフィックスを付与
- 関連ノートとのリンクを追加

#### 2. タイトル改題（15分）
- 3～5個のノートについて、タイトルを改題
- 内容の不正確な部分を修正
- 関連記事への参照を追加

#### 3. フィルター設定の見直し（15分）
- 今週の学習テーマに応じて、フィルター設定を変更
- 不要な情報を非表示に
- 新しいワークスペースの必要性を検討

### 要点
**15分で「完璧な整理」を目指さない**。

その代わり、「週2～3回、少しずつ改善」というアプローチ。

## 長期的な成果

### 1年続けた場合
```
15分 × 2.5回/週 = 37.5分/週
37.5分 × 52週 = 1,950分/年 ≒ 32.5時間/年

この32.5時間で：
- 数百個のノートが体系化
- 複数カテゴリの知識が統合
- 自分だけの知的生産システムが構築
```

### 5年続けた場合
```
32.5時間 × 5年 = 162.5時間

この時間投資で：
- 複数の専門領域が習得可能な水準に
- 生涯の知識資産が形成
- 新しい学習が飛躍的に効率化
```

## 挫折を防ぐコツ

### 無理しない
- 「今週は1回だけ」でもOK
- 体調が悪い週は休む
- 「週2～3回」は目安であり、必須ではない

### 興味の在処で判断
- 気が乗らないカテゴリは後回し
- 今興味あることだけやる
- これが継続性につながる

### 柔軟性を活用
- ワークスペース切り替えで、やることを限定
- 「今日はミュージックだけ」という絞り込み
- 認知負荷を下げることで、続きやすく

## RINK哲学の実現

「生涯をかけて少しずつ育てる趣味」という考え方が、ここで具現化される。

- 毎日やる必要はない
- 無理なく続ける
- それでも5年、10年で大きな資産が形成される

これが、単なる「ノート管理ツール」ではなく、**生涯の知的生産パートナー**としてObsidianを使える理由。
