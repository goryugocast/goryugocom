## 概要

Base プラグインにおける最大のUX上の強みは、**フィルター条件をUI上の直感的操作のみで設定できる**こと。

## DataViewとの操作の違い

### DataView
```javascript
// コード記述が必須
WHERE date >= today AND status != "完了"
```

修正する際：
- コードを編集
- シンタックスエラーのリスク
- 動作確認が手間

### Base
1. ポチポチ操作でフィルター追加
2. プルダウンで演算子選択
3. 値を入力
4. **即座に結果反映**

## UI操作の流れ

1. **フィルター追加**: ボタン１クリック
2. **条件選択**: プロパティを選択
3. **演算子選択**: 「=」「>」「starts with」など
4. **値入力**: テキストボックスに入力
5. **プレビュー**: 同時に結果表示

## 修正時の素早さ

DataViewでコード修正する場合の手間が大幅削減：

```base
# 最初の絞り込み
prefix: "M.C"

# ユーザーが直接数字を変更
limit: 5
↓
limit: 10
```

- フィルター条件の数字を直接編集
- コード記述不要
- エラーのリスク激減

## パラメータの可視化

「バッククォート3つ+base」で記述した場合：

- UI上で現在のフィルター条件が表示される
- 何を指定しているか一目瞭然
- ドキュメント記法を見て学習可能

## 継続使用へのハードルの低さ

直感的UI → 継続使用 → 習得定着 のサイクルが自然に形成される。
