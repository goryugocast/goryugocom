# 読書メモ用のフォルダ構造

## 概要

Obsidianでの読書メモ管理では、基本的にフォルダを細かく分けすぎず、シンプルな構造を維持することが重要。読書メモは専用フォルダ（Books）に集約し、年ごとにサブフォルダを作成する程度のシンプルな構造が使いやすい。

## 推奨されるフォルダ構造

```
Obsidian Vault/
├── assets/          # 画像などのファイル
├── projects/        # プロジェクト関連
├── daily/           # 日記
├── notes/           # アトミックノート（基本的に階層化しない）
└── books/           # 読書メモ専用フォルダ
    ├── 2021/
    ├── 2022/
    └── ...
```

## 重要な原則

### 整理しすぎない
- 読書メモは基本的にbooksフォルダ直下に置く
- 年ごとの分類のみ行う（月別は細かすぎて管理が煩雑になる）
- 細かいカテゴリ分けはフォルダではなく、リンクとタグで行う

### 特別なものだけ分ける
- 趣味的な本（漫画、小説など）で、後から一覧で見返したいものは別フォルダにしてもよい
- ただし、分けるのは例外的な場合のみ

## 実践上の注意点

### iPhoneでの閲覧制限
- ファイル数が多くなりすぎると、iPhoneのファイルアプリで開けなくなることがある
- 年ごとに分けることで、ある程度この問題を緩和できる
- ただし、iPhone上でファイルアプリから直接開く必要性は低い

### 検索とリンクで管理
- フォルダ構造に頼らず、検索機能とノートリンクで本を見つける
- 本のリストノート（MOC）を活用する

## 実践のヒント

- フォルダ移動はショートカットキーを割り当てておくと便利（Move file to another folder）
- 読み終わった本をbooksフォルダに移動させる習慣をつける
- 複雑な分類が必要になったら、その時点で考える（事前に構造を作りすぎない）
