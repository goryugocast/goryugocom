# 日付ベースから意味重視へシフトするファイル命名規則

## 概要
従来の`20251113_...`という日付ベースの命名規則から、`ks.rem.040_🔔`という意味重視の規則へシフトしている。これはAI時代の検索・管理効率を考慮した進化。

## 従来の問題点

### 日付ベースの課題
```
20251113_セミナー直前リマインド.md
20251120_セミナー告知.md
```

- 検索時に日付で探すことが少ない
- AIに指示する際、日付の更新が手間
- ファイルの意味が一目でわかりにくい

## 新しい命名規則の構成

### 形式
```
ks.rem.040_🔔セミナー直前リマインド.md
```

### 各要素の役割

1. **接頭辞（ks.rem.040）**
   - `ks` = プロジェクトコード（Knowledge Stuck）
   - `rem` = ファイルの種類（reminder）
   - `040` = 連番
   - ドットで区切ることでAI・ターミナル対応

2. **区切り文字（_）**
   - スペースの代わりにアンダースコアを使用
   - ターミナルで問題になるスペースを回避

3. **絵文字（🔔）**
   - 直感的な識別性を高める
   - 視覚的に素早く目的のファイルが見つかる

4. **説明的な名前**
   - ファイルの内容が一目瞭然
   - 人間にもAIにも理解しやすい

## メリット

- **AI活用が容易** - AIによるバッチ処理が簡単
- **ターミナル対応** - スクリプトやコマンドラインでも扱いやすい
- **クイックスイッチャー最適化** - `ks.rem`だけで目的ファイルに素早くアクセス
- **視認性** - 絵文字で瞬間的に識別可能

## 移行のポイント

- 日付管理は`date_published`をフロントマターで
- ファイル名は「何か」を優先
- 連番は複製時に自動的に更新される仕組みを構築
