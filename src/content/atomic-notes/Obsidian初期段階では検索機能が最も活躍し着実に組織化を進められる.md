# Obsidian初期段階では検索機能が最も活躍し着実に組織化を進められる

Obsidianを新しく導入したばかりの時期は、必ずファイル整理が追いつきません。この段階では、検索機能が最も活躍する時期です。正確な運用戦略を持つことで、混乱を避けながら着実に組織化を進められます。

## 初期段階の特徴

- タイトルがきちんとついていないノートが多い
- Cmd/Oコマンドパレットで見つからないことが頻繁
- デイリーノートに情報が散在している
- 分類体系がまだ完成していない

## この段階での検索戦略

### ステップ1：デイリーノートに全て集約
とにかく「何でもいいからデイリーノートに書く」という原則を徹底します。

### ステップ2：キーワードで必要な情報を検索
```
path:daily/ キーワード
```
デイリーノート内から必要な情報を素早く抽出。

### ステップ3：関連情報を一覧で把握
Copy search resultsで関連ノートを自動抽出し、トピックノートを構築。

### ステップ4：整理のラインを引く
ここまでのプロセスを通じて「これは整理ノートが必要」な領域が見える化されます。

## デイリーノート活用の鍵

デイリーノートは「何でも入れてOK」という位置づけが重要です。これは「整理を強制しない」ため：

- 思いついたことを即座に記録
- あとで分類・整理する前提で蓄積
- 検索で必要な時に取り出す

## 整理完成への移行

ノート数が100、200と増えてくると、「について」という命名規則が効き始めます。つまり：

1. **初期（0-50ノート）**: キーワード + `path:daily/` で検索
2. **中期（50-200ノート）**: 「について」で整理済みトピックノートを探す
3. **完成期（200+ノート）**: ほぼ検索不要。Cmd/Oで直接アクセス
