# Baseプラグインのフィルター機能で認知負荷を軽減

## 概要

情報量が増えてくると、すべてを一度に表示するのは非効率。**Baseプラグインのフィルター機能**を使い、「**今見るべき情報**」を動的にコントロールすることで、認知負荷を大幅に軽減できる。

## フィルターの実例

### ギター関連ノートを非表示にする場合

```
フィルター: does not start with G
（Gで始まらないものだけ表示）
```

効果：
- ギター関連のノート（G.1.a.1など）が一時的に非表示
- 画面上の情報量が削減
- 集中したいカテゴリに専念可能

### 複数フィルターの組み合わせ

```
フィルター1: does not start with G （ギター除外）
フィルター2: does not start with AL （Ableton Live除外）
フィルター3: does not start with P （プラグイン除外）
```

## 実装のメリット

### 1. 認知負荷の最小化
- 「今は音楽理論に専念する」→ギター関連を非表示
- 「今はAbleton Liveの設定を詳しく学ぶ」→他のジャンルを非表示

### 2. 現在地の把握
- 情報量が減ると、全体を俯瞰しやすい
- 「今どの領域を学んでいるか」が明確

### 3. モード的な運用
- その週の学習目標に合わせてフィルター設定を変更
- 柔軟な学習管理

## フィルター設定の変更

### 復活させるとき
- 「ギターを再び学び始めたので表示する」
- フィルターを削除、またはパラメータを変更

### 注意点
- あくまで「非表示」であり、データは削除されない
- いつでも復活可能

## RINK運用での活用

大規模なノート群（200+項目）を管理する場合、フィルターなしには全体の把握が困難。フィルター機能により、**柔軟に焦点を絞りながら、全体構造は保持**できる。

これが、「**いつでもやり方を変えられる**」というRINKの柔軟性の実現を支える重要な機能。
