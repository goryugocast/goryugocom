# 検索対象を1～2個のプロパティに絞る実装

## 概要
Another Quick Switcherで検索対象を増やしすぎると、クイックスイッチャーの本来の「素早さ」が失われる。1～2個のプロパティに限定することで、バランスの取れた検索体験を実現。

## 複雑化の落とし穴

### 多くのプロパティを検索対象にした場合
```
検索対象：
- people（人物）
- project（プロジェクト）
- tags（タグ）
- author（著者）
- publisher（出版社）
- genre（ジャンル）
```

**問題点**:
- ノイズが増える
- クイックではなくなる
- 設定を覚えられない

## 推奨構成

### 最小限（1個）
```
people プロパティのみ
```
日常業務の80%をカバー可能。

### 実用的（2個）
```
1. people（人物）- 人とのやり取り検索
2. project（プロジェクト）- プロジェクト単位での検索
```

## 実装パターン

### パターン1：デフォルト検索を拡張
```
通常のCmd+O:
- ファイル名
- people プロパティ
```

### パターン2：カスタムコマンドで人物検索
```
コマンド: @people
例）@うりなみ → うりなみさんと関連するノートすべて
```

### パターン3：カスタムコマンドでプロジェクト検索
```
コマンド: #project
例）#seminar → seminar関連のファイルすべて
```

## バランスの取り方

### クイックさを保つ条件
- 覚えやすいコマンド（2～3個まで）
- プロパティ値の表記が統一されている
- 実際によく使うプロパティだけに限定

### 実装時のチェックリスト
- [ ] 通常検索で必要な情報は満たされるか
- [ ] 新しく追加するプロパティは本当に必要か
- [ ] 複数のプロパティを同時に検索することがあるか
- [ ] コマンドの数を3個以下に保てるか

## 段階的な追加戦略

### 最初
```
デフォルト: people プロパティのみ
```

### 必要に応じて追加
```
カスタムコマンド: project プロパティ用
```

### それ以上は...
```
左側のサーチパネルを使う
（Another Quick Switcherの本来の役割ではない複雑検索）
```

## 注意点

相川さんのプラグインは豊富な設定がある故に、つい複雑化してしまいやすい。設定画面を見ると「いろいろできそう」という誘惑に駆られるが、実用性を優先することが長期的な効率につながる。
