# 毎月セミナーの関連ファイルをtopicで一元管理

## 概要
毎月開催するセミナーの関連ファイル（告知、リマインド、アーカイブ、資料）を、共通の`topic`プロパティでフロントマターに記入し、Baseプラグインで一元管理する。

## 問題設定

### セミナー運用での課題
```
告知ファイル
リマインドメール案
アーカイブ配信スクリプト
資料ファイル
事後アンケート
スライド
...
```
複数のファイルが関わるため、全体を把握するのが大変。

## 解決策：topicプロパティで一元管理

### ファイル構成例

#### 告知ファイル
```yaml
---
topic: seminar040
type: announcement
date_published: 2025-09-13
---
# セミナー40回目の告知
```

#### リマインドメール
```yaml
---
topic: seminar040
type: reminder
date_published: 2025-10-01
---
# セミナー40回目のリマインド
```

#### アーカイブ配信
```yaml
---
topic: seminar040
type: archive
date_published: 2025-10-15
---
# セミナー40回目アーカイブ配信スクリプト
```

#### 資料・スライド
```yaml
---
topic: seminar040
type: material
date_published: 2025-10-08
---
# セミナー40回目の参考資料
```

## Base クエリで一元表示

### シンプルなクエリ
```
topic: seminar040
```
セミナー40に関連するすべてのファイルを表示。

### より詳細なクエリ
```
topic: seminar040
sort by date_published asc
```
セミナー40の全ファイルを公開順に表示。

```
topic: seminar040
type != archive
```
セミナー40のうち、アーカイブ以外を表示（進行中のセミナー情報）。

## ファイル命名規則との組み合わせ

### 統一された命名規則
```
ks.announce.040_セミナー告知
ks.remind.040_セミナー直前リマインド
ks.archive.040_セミナーアーカイブ配信
```

### プレフィックスでも、topicでも検索可能
- **プレフィックス検索**（ファイル一覧から）：`ks.remind.040` と入力
- **topic検索**（Base クエリから）：`seminar040` で関連ファイル一覧
- **Another Quick Switcherで**：`@seminar040` で関連ファイル表示

多元的なアクセス方法が可能。

## Base クエリの活用例

### セミナー40の進行ボード

```markdown
## セミナー40 運用チェック

### 準備フェーズ
| ファイル | 状態 |
| --- | --- |
| 告知 | 完了 |
| スライド | 作成中 |
| 参考資料 | 完了 |

### 実施・配信フェーズ
Base クエリで topic: seminar040 を表示
```

すべての関連ファイルをBase経由で参照可能。

## タイプの分類例

```yaml
type:
  - announcement（告知）
  - reminder（リマインド）
  - archive（アーカイブ配信）
  - material（参考資料）
  - feedback（事後アンケート）
  - slides（スライド）
  - script（進行スクリプト）
```

セミナーの各段階に応じたファイルを分類できる。

## 複製ワークフローとの統合

### セミナー39→40への複製

```yaml
# セミナー39（完成）
---
topic: seminar039
type: archive
---

# 複製してセミナー40を作成
---
topic: seminar040
type: announcement
date_published: 2025-11-13
---
```

topic番号を更新するだけで、プロパティ構造が引き継がれる。

## メリット

1. **関連ファイルの即座把握**
   - Base で `topic: seminar040` と検索するだけで全ファイル表示

2. **プロジェクトノートの自動化**
   - 手動でリンク張る必要がない
   - Baseが自動集約

3. **前回の改善を反映**
   - セミナー39で構築した topic 分類が、セミナー40でも継承される

4. **新人スタッフのオンボーディング**
   - 「セミナー40に関連するすべてのファイルが見たい」という要望に、一瞬で応じられる
