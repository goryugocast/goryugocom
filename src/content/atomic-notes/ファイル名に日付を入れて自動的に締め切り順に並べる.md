# ファイル名に日付を入れて自動的に締め切り順に並べる

## 概要
タスクノートのファイル名に日付（例：250826）を含めることで、Obsidianの自動ソート機能により、締め切り順に自動的に並べられるシステム。追加のツールやプラグイン不要で実現できる実践的な工夫。

## ファイル名の構成例
```
250826 📨BC121 執筆
250828 🎤ポッドキャスト編集配信
250821 📝セミナー招待記事
```

## 動作原理

### Obsidianのファイルソート機能
- デフォルトでは「Files」のソート方法を「A to Z」（アルファベット順）に設定
- ファイル名の最初に日付を付けることで、数字順にソートされる
- 「250821」→「250826」→「250828」と自動的に並ぶ

### なぜこれが有効か
- 理論的には上から順番に全てを終わらせれば、締め切りを過ぎることがない
- 優先順位の手動管理が不要
- ファイルリストを見るだけで、次にやるべき仕事が明確

## ファイル名の工夫

### 日付フォーマット
- YYMMDD（例：250821）形式が最適
- 年月日順で自動ソートされるため、時系列が保たれる

### 日付の後の情報
- 絵文字でカテゴリを表示（📨メール、🎤音声コンテンツなど）
- 簡潔な説明を付けることで、リストを見るだけで内容が理解できる

## 実装時の注意点

### 複数のプロジェクト
- 複数のシリーズがある場合、親フォルダで分ける
- あるいはファイル名の最後にプロジェクト名を付ける

### 過去のタスク
- 完了後は別フォルダに移すか、アーカイブする
- 現在のフォルダには「活動中のタスク」だけを保つ

### 実行順序
- 常に一番上のタスクから開始すると効率的
- やむを得ず順序を変える場合は、ファイル名を編集して順序を変更

## メリット

### 自動管理
- 追加の作業なしに優先順位が決まる
- 判断疲れ（decision fatigue）が軽減される

### 視認性
- デイリーノートを見なくても、ファイルリストから今日の仕事が分かる
- スクロール不要で全体像が見える

### 柔軟性
- ファイル名を編集するだけで、順序や期限を変更できる
- テンプレートや複雑な設定が不要
