# Another Quick Switcherでプロパティも検索対象にする

## 概要
Another Quick Switcherプラグイン（相川さん作成）を使用することで、クイックスイッチャー（Cmd+O）の検索対象をファイル名だけでなく、フロントマターのプロパティにも拡張できる。

## 標準クイックスイッチャーの制限

```
標準機能: ファイル名のテキストマッチのみ
→ ファイル名に「著者名」がなければ、著者で検索できない
```

## Another Quick Switcherの拡張機能

### プロパティを検索対象に含める
```
著者: 小川仁志
```
というプロパティを持つノートを、ファイル名に著者名がなくても「小川仁志」で検索可能に。

### カスタムコマンドの作成例

#### 人物検索用カスタムコマンド
```
prefix: @
search properties: people
```
`@うりなみ`で、people プロパティに「うりなみ」を含むノートをすべて表示。

#### プロジェクト検索用カスタムコマンド
```
prefix: #
search properties: project
```
`#seminar040`で、関連するセミナーファイルをすべて表示。

## 実装時の注意点

### 複雑化を避ける
相川さんのプラグインはマニアック設定が豊富だが、最初は1～2個のシンプルなコマンドに限定。
```
1. 通常検索（ファイル名+プロパティ）
2. 人物検索用のカスタムコマンド
```

### 実用的な設定例

```
【推奨設定】
- デフォルト: ファイル名 + people プロパティ
- 追加: 1個だけカスタムコマンド

【非推奨】
- 5個以上のカスタムコマンド
- 複雑なフィルター条件
```

## メリット

1. **ファイル名に情報を詰め込まない**
   - ファイル名はシンプルに
   - 詳細情報はプロパティで

2. **検索スピードの向上**
   - 著者、人物、プロジェクトなど複数の軸で検索可能

3. **管理コストの削減**
   - プロパティの一貫性さえ保てば、検索精度が自動的に向上

## vs. 左側のサーチパネル

- **Another Quick Switcher**: 素早く目的のファイルにアクセス
- **左側サーチパネル**: 何か見つからないという緊急時の最終手段

実務では、Another Quick Switcherで大抵のファイルが見つかる状態を目指す。
