# アンダースコアをスペースの代わりに使う

## 概要
ファイル名にスペースを含めるのではなく、アンダースコア（_）で代替することで、ターミナルやAIとの相性を大幅に向上させる。

## スペースの問題点

### ターミナルでの扱いづらさ
```bash
# スペース含むファイル名は引用符が必須
cat "my file name.md"

# アンダースコアなら不要
cat my_file_name.md
```

### AI指示時の複雑性
スペア含むファイル名をAIに指示すると、クオート処理が必要になるなど指示が複雑化。

### 文字化け・エンコード問題
特に日本語や特殊文字とスペースが混在すると、エンコーディング問題が発生しやすい。

## アンダースコアの利点

### 見た目と機能性のバランス
```
ks.rem.040_🔔セミナー直前リマインド.md
```
- スペースと見た目は似ている
- しかし技術的には問題がない
- 可読性を損なわない

### 一貫性の維持
プレフィックスはドット、説明部分はアンダースコアという使い分けで、視覚的な階層構造が生まれる。

## 実装例

### ダメな例
```
ks.rem.040 セミナー直前リマインド.md
```

### 良い例
```
ks.rem.040_セミナー直前リマインド.md
ks.rem.040_🔔セミナー直前リマインド.md
```

## 日本語環境での特に重要性

日本語テキストを含む場合、スペースは半角全角の問題が生じやすい。アンダースコアに統一することで、このリスクを完全に排除できる。
