# エバーグリーンノートのタイトルはAPIのようなもの

## 概要

エバーグリーンノートの概念を提唱したアンディ・マツチャック（Andy Matuschak）によれば、ノートのタイトルは「API」のようなものであるべき。つまり、タイトルを見ただけで、そのノートが何を提供するのか（どんな情報や機能を持っているのか）が明確に分かるべきだという原則。

## APIの比喩の意味

### プログラミングにおけるAPI
- API（Application Programming Interface）は、他のプログラムから呼び出せる機能の仕様
- APIの名前を見れば、それが何をする機能なのかが明確に分かる
- 使う側は内部実装を知らなくても、名前から用途を判断できる

### ノートタイトルへの応用
- タイトルを見ただけで、そのノートの内容や目的が分かるべき
- 他のノートからリンクする時に、タイトルで判断できるべき
- 内容を開かなくても、タイトルから価値が分かるべき

## 実践的な意味

### 明確で具体的なタイトル
- 曖昧な表現を避ける
- 具体的な主張や定義を含める
- 誰が見ても同じ理解ができるタイトルにする

### 再利用可能性
- 他のノートから参照しやすいタイトルにする
- プロジェクトや文脈が変わっても、タイトルの意味が変わらない
- 長期的に価値を持ち続けるタイトルにする

## プログラマーでない人への応用

- APIの概念を知らなくても、「タイトルは機能の説明」と考えればよい
- 「このノートは何をしてくれるのか」「何を教えてくれるのか」を明示する
- 道具箱の引き出しのラベルのように、中身が一目で分かるようにする
