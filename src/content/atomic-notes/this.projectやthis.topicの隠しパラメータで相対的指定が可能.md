## 概要

Base プラグインの隠しパラメータ（`this.project`、`this.topic`など）により、**相対的・動的な指定が可能**になり、テンプレートの汎用性が飛躍的に向上する。

## 従来のアプローチ

フィルター条件が固定値で指定される：

```
project = "BookCatalyst"
project = "ナレッジスタック"
```

- 異なるプロジェクトでは使い回せない
- 各プロジェクト用に個別カスタマイズが必須

## this パラメータの力

```
project = this.project
topic = this.topic
```

- **相対的指定**: 「このノート自身のプロジェクト」という指定
- コピー&ペーストで他のファイルに貼付可能
- その場所の文脈に応じた内容が自動で表示

## テンプレートの再利用性

1. **一度テンプレートを作成**
   - 複数の相対パラメータを設定

2. **他のノートにコピー&ペースト**
   - 自動的に文脈が適用される

3. **追加カスタマイズは最小限**
   - 基本的には設定不要

## JavaScriptオブジェクト的思想

JavaScriptの`this`キーワード同様、「**現在の実行コンテキスト**」を参照する概念。

- 高度だが非常に強力
- 一度理解すると多くの応用が可能

## 実例

- `this.people`：このノート担当者の記録
- `this.topic`：同一トピック内の関連ノート
- `this.project`：同一プロジェクト内のタスク
