# 未来の日付のデイリーノートで予定を事前準備する

## 概要
Obsidianのデイリーノート機能を使って、未来の日付のノートをあらかじめ作成し、旅行やイベント、チケット発売日など、事前準備が必要な予定を書き込むアプローチ。

## 未来のデイリーノート活用

### 旅行の計画
```
2025-09-15: 京都旅行（3日目）
- 伏見稲荷大社の参拝
- 帰路の準備
- 帰宅
```

### チケット発売日の準備
```
2025-09-18:
- 14:00 コンサートチケット発売開始
- あらかじめ支払い情報を用意しておく
```

### 重要な期日の事前注記
```
2025-10-01:
- セミナー申し込み締め切り前日（明日が締め切り）
- 参加者確認メールを送信
```

## メリット

### 準備時間の確保
- 未来の予定を「今から」準備できる
- 前日の夜に「明日からチケット販売」と書いておくと、忘れない
- バタバタした朝に慌てることが減る

### ログと予定の統一管理
- 旅行期間中のログと、旅行に関する予定が同じファイルに記録される
- 「この日は何をしていたか」と「この日の予定は何だったか」が同時に確認できる
- 別途カレンダーアプリを開く必要がない

### 視認性の向上
- カレンダーアプリと異なり、テキストベースで詳細が記述できる
- チェックリストを作成すれば、旅行の持ち物確認も可能
- 予定に関する詳細メモを同じところに集約できる

## 実装方法

### デイリーノートの作成
1. Cmd/Ctrl + D で今日のノートを開く
2. あるいはコマンドパレットで「Create note for specific date」を選択
3. 未来の日付を指定（例：2025-09-15）
4. ノートを作成して、予定を記入

### 記述方法
- 時間帯を明記（「18:00 チケット販売開始」）
- 必要な準備事項を箇条書き
- 関連ノートへのリンク（旅行ノート、チケット情報など）

## 実例

### チケット発売日の前夜
```
2025-09-17（木）
- 明日からコンサートチケット発売開始（14:00～）
- 支払い情報を確認してメモ帳に記録
- クレジットカード情報は別管理
- 発売サイトのログイン情報を確認
```

### 翌日（発売日当日）
```
2025-09-18（金）
- 14:00 コンサートチケット発売開始（ここに書いてあった！）
- 14:05 購入完了
- 確認メールが届いた
```

## 複数日にわたる予定への対応

### 旅行の例
```
2025-09-15
- 09:00 家を出発
- 12:30 京都到着
- 15:00 ホテルチェックイン
- 19:00 夕食

2025-09-16
- 08:00 朝食
- 09:30 伏見稲荷大社参拝
- 14:00 貴船神社参拝
- 18:00 夕食

2025-09-17
- チェックアウト
- 新幹線で帰路
- 帰宅
```

各日付のデイリーノートに予定を記述することで、旅行全体が複数のノートに分散される。後から振り返る際も、日付ごとに内容が分かる。

## カレンダー機能との違い

### Obsidianデイリーノートの利点
- テキストベースで詳細が書ける
- Markdownで自由に整形できる
- 関連ノートへのリンクが貼れる
- ハッシュタグで検索可能

### 従来のカレンダーアプリとの比較
- カレンダーアプリは時間軸に最適化
- デイリーノートはコンテンツ量に最適化
- 目的に応じて使い分けるのが理想的

## 注意点

### 実行のルール
- 未来のノートを作成しても、実行時に更新する習慣をつける
- 計画だけで実行しない場合、ログ記録が抜ける

### 変更への対応
- 予定が変わった場合は、あらかじめ作成したノートを編集
- 削除せず「変更あり」と追記することで、変更履歴が残る

### 過度な準備の回避
- 全ての日付のノートを事前作成する必要はない
- 「準備が必要」な予定だけ対象にする
