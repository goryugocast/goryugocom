# GitとGitHubの役割と連携

## 概要
Gitは分散型バージョン管理システムであり、ファイルの変更履歴を管理する「プログラム」そのものである。一方、GitHubは、Gitで管理されるデータの「本体（リポジトリ）」をクラウド上で提供するサービスであり、この二つが連携することで、効率的なバージョン管理と共有が可能になる。

## 重要なポイント
- **Gitの機能**:
    - ファイルの変更履歴を記録し、過去の任意の時点の状態に戻すことができる。
    - データの「分散型」管理に優れており、各ユーザーがローカルにコピーを持ち、それぞれが変更を加えられる。
- **GitHubの機能**:
    - Gitリポジトリをホスティングするクラウドサービス。これにより、個人でサーバーを用意することなく、Gitの機能を活用できる。
    - オープンソースプロジェクトへの参加を容易にしたり、複数人での協調作業をサポートする様々な機能（Issue, Pull Requestなど）を提供する。
- **連携による利便性**: 個人でGitを使う場合、GitHub DesktopのようなGUIツールとGitHubサービスを組み合わせることで、コマンドライン操作なしに手軽にバージョン管理の恩恵を受けられる。

## 実践のヒント / 関連する概念
- GitとGitHubの違いを理解することで、それぞれのツールの得意分野を活かしたデータ管理が可能になる。
- GitHub Desktopは、Gitの複雑な概念を意識することなく、直感的な操作でバージョン管理を行えるため、初心者にも推奨される。
