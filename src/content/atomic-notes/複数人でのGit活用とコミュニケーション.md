# 複数人でのGit活用とコミュニケーション

## 概要
Gitは複数人でのプロジェクトにおいて、ファイルの変更履歴管理や衝突解決だけでなく、チーム内のコミュニケーションを円滑にするツールとしても機能する。特に、作業の進捗を「押しつけがましくなく」共有できる点が、協調作業における適度な緊張感とモチベーション維持に貢献する。

## 重要なポイント
- **「押しつけがましくない進捗報告」**: 各自がコミットするたびに残されるメッセージは、相手に「今何をやっているか」をさりげなく伝える進捗報告となる。これにより、定期的な進捗会議とは異なる、緩やかな情報共有が実現する。
- **適度な緊張感とモチベーション**: 他のメンバーのコミット履歴を見ることで、「自分もちゃんとやらなければ」という良い意味でのプレッシャーが生まれ、個人のモチベーション維持に繋がる。
- **変更内容の明確化**: 誰が、いつ、どこを変更したのかが一目瞭然となるため、複数人が同時にファイルを変更しても、意図しない上書きを防ぎ、衝突発生時の解決も容易になる。

## 実践のヒント / 関連する概念
- 夫婦で共同執筆するようなプロジェクトにおいて、Gitを使うことで互いの進捗を把握し、スムーズな連携が可能になる。
- IssueやPull RequestといったGitHubの機能と組み合わせることで、より体系的なプロジェクト管理とコミュニケーションを実現できる。
