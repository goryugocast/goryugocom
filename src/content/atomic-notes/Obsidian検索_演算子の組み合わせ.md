# Obsidian検索：演算子の組み合わせ

複数の検索演算子を組み合わせることで、複雑な検索ニーズに対応できます。初心者は `path:`、`file:`、`-` の3つに絞ることで、実践的な成果を得やすくなります。

## 基本演算子の組み合わせ

### AND（暗黙的な結合）
```
path:daily/ ドミナント
```
デイリーノートかつドミナントを含むノート。スペースで自動的に AND 結合されます。

### OR（複数キーワード）
```
ドミナント OR セカンダリー
```
ドミナントまたはセカンダリーのいずれかを含むノート。複数の関連概念を同時に検索。

## 実践的な複合検索の例

### 例1：整理状況の診断
```
path:daily/ ドミナント -file:weekly
```
- デイリーノート内から
- 「ドミナント」を含みながら
- ウィークリーは除外

### 例2：読んだ本から引用を検索
```
path:books/ 遺伝子 -file:reference
```
- 読書記録フォルダから
- 「遺伝子」という言葉を含みながら
- 参考資料ファイルは除外

### 例3：複数トピックの統合検索
```
(ドミナント OR セカンダリー) -path:daily/ file:music
```
複数の音楽概念をまとめて検索し、整理済みノートだけを表示。

## 注意点

- スペルは正確に。AIの「Explain search term」機能で確認可能
- 複雑すぎる検索より、シンプルで使いこなせる検索を優先
- 結果が多すぎる場合は、除外条件を追加して絞り込む
