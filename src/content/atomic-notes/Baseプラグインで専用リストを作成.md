## 概要

Baseプラグイン（旧Dataview）を使用することで、フロントマターの条件に合致するノートを自動抽出し、専用の一覧ページを動的に生成できます。

## Baseプラグインの基本

### インストール
- Obsidian Marketplaceから「Base」で検索
- またはコミュニティプラグイン設定から追加

### 基本的な使用方法
- マークダウンのコードブロックに専用構文を記述
- ノート保存時に自動的に一覧が生成される

## プロジェクト別リスト作成例

### 音楽プロジェクト用ノート

#### ファイル名
`音楽プロジェクト一覧.md`

#### 内容
```
# 音楽プロジェクト

以下は`project: music`が設定されたノートの一覧です。

\`\`\`
project = "music"
\`\`\`
```

### 仕事案件用ノート

```
# ClientA プロジェクト

## 進行中のタスク

\`\`\`
project = "clientA" and status != "completed"
\`\`\`

## 完了済みタスク

\`\`\`
project = "clientA" and status = "completed"
\`\`\`
```

## 高度なクエリ例

### 複数条件の組み合わせ
```
project = "music" and tags contains "機材" and updated > date(2025-01-01)
```

### プリオリティ付きフィルタ
```
project = "music" and priority = "high"
```

### 更新日でのソート
```
project = "music" sort by updated descending limit 10
```

## Baseプラグインの利点

### 動的な一覧生成
- ノートを追加・変更すると自動更新
- 手動メンテナンス不要
- 常に最新の状態を表示

### 複数条件での検索
- AND / OR 演算子対応
- 比較演算子（>、<、=）対応
- 日付や数値での絞り込み

### プロジェクト管理の効率化
- 関連ノートをワンクリックで全表示
- スコープを意識した整理が容易
- チーム共有時の可視性向上

## Publishでの制約と対応

### 問題点
- Publish（パブリック共有）ではBase結果が表示されない
- 理由：Publishは静的HTML生成
- Baseは動的生成の仕様

### 対策1：手動でリスト作成
- Publishページには手動でリンク集を作成
- 厳選した内容を提示
- 200個より10個の方が価値が高い

### 対策2：複数ページでの対応
```
- [[音楽機材レビュー]]
- [[音楽制作テクニック]]
- [[楽器別ガイド]]
```

## 実装のコツ

### ステップ1：メタデータ設計
- `project:`フィールドを統一
- `status:`フィールドも活用
- `priority:`などの追加も検討

### ステップ2：クエリの試行錯誤
- 小さなクエリから始める
- 段階的に条件を追加
- エラーメッセージから学ぶ

### ステップ3：定期的な見直し
- 月1回はクエリを確認
- 新しい条件が必要か検討
- アーカイブの仕組みを整備

## 注意点

### Baseプラグインの限界
- 非常に複雑な検索は苦手
- パフォーマンスが落ちる場合も
- 正規表現には対応していない

### メタデータ入力の大切さ
- クエリの精度 = メタデータの品質
- 最初の入力が重要
- テンプレート化して自動化

## ベストプラクティス

### 1段階型プロジェクト管理
```
基本フォルダ/
├── 進行中プロジェクト一覧.md
├── プロジェクトA/
│   ├── プロジェクトA全体.md（Base埋め込み）
│   └── ノート群
└── プロジェクトB/
```

### 2段階型システム
```
1. Base で動的に一覧生成（内部用）
2. Publish 用に手動でまとめ直す（外部用）
```

Baseプラグインの活用により、大規模ノートベースでも全体把握が容易になります。
