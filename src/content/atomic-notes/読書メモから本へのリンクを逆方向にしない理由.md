# 読書メモから本へのリンクを逆方向にしない理由

## 概要

Note Refactorプラグインでアトミックノートを作成する際、新しいノートには`From: [[元の本のタイトル]]`というテキストを残すが、あえてWikiリンク形式にせず、リンクを貼らないようにしている。理由は、本のノートに対するバックリンクが多くなりすぎて、本当に重要なリンクが埋もれてしまうため。

## リンクの方向性

### 一方向のリンク
- 本のノート → アトミックノート（リンクあり）
- アトミックノート → 本のノート（リンクなし、テキストのみ）

### リンクを貼らない理由
- 一冊の本から数十個のアトミックノートを作ると、本のノートへのバックリンクが膨大になる
- バックリンクが多すぎると、本当に関連性の高いノートが見つけにくくなる
- 逆方向（アトミックノート→本）は、グラフビューで確認できれば十分

## 実装方法

### Note Refactorの設定
フロントマターやテンプレートで以下のように設定：
```
From: 本のタイトル
```
リンク記法 `[[]]` を使わず、プレーンテキストで記述。

### メリット
- 本のノートがバックリンクで溢れない
- 本当に関連性の高いノート（他のアトミックノートからのリンク）が見えやすい
- 必要な時はグラフビューで元の本を確認できる

## 考え方のポイント

### バックリンクの質を保つ
- 自動的に生成されるリンクで溢れないようにする
- 意図的に貼ったリンクだけが見えるようにする
- ノイズを減らし、シグナルを強調する

### 情報は逆方向からも確認できる
- アトミックノートには出典情報が残っている
- グラフビューを使えば関係性は見える
- 双方向リンクが必ずしも必要とは限らない

## 実践のヒント

- 本のノートから、重要なアトミックノートへは積極的にリンクを貼る
- アトミックノート同士のリンクを充実させる方が重要
- 出典情報は検索できればよいので、リンクでなくてもよい
