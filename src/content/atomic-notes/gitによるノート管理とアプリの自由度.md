# gitによるノート管理とアプリの自由度

## 概要
Obsidianのデータをgitで管理することは、単なるデータ同期手段にとどまらず、使用するアプリケーションの選択肢を広げ、データのポータビリティと管理の自由度を大幅に向上させる。

## 重要なポイント
- **Obsidian Syncからの解放**: 公式のObsidian Syncを使わずとも、gitを介してデータを同期できる。これにより、同期の遅延やファイル重複といった問題から解放される可能性もある。
- **エディタの選択肢拡大**: gitで管理されたMarkdownファイルは汎用性が高く、Working Copyなどのアプリを介することで、iA WriterなどObsidian以外の好きなエディタで自由に編集できるようになる。
- **データのポータビリティ**: データがgitリポジトリとして管理されるため、特定のアプリに依存することなく、いつでもどこでも自分のノートにアクセス・編集が可能になる。
- **作業の振り返り**: gitのコミット履歴は、日々の作業内容を振り返るための記録としても活用でき、自己分析や生産性向上に貢献する。

## 実践のヒント / 関連する概念
- iOSデバイスではWorking Copyアプリを介してGitHubと連携させることで、他のエディタからもgit管理下のファイルにアクセスできるようになる。
- 「このiPhone内」のファイルをWorking Copyと同期させることで、ObsidianやLogseqのデータをgitで管理することも可能になる。
