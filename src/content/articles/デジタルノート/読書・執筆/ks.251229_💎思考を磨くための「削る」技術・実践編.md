## ✨ 生成AI時代に「削る技術」が価値になる理由

「削る技術」とだけ聞くと、多くの人が「要約する力」や「短くする技術」のことだと思うかもしれません。ですが、削る技術の本質は、**「何を捨てるか」を自分に問い続けること**にあります。

確かに要約する力や短くする技術も含まれてはいるのですが、何を残して、何を捨てて削っていくのかという部分は人間である私達が決めて伝えないといけない部分です。

生成AIが普及してから、文章やアイデアを増やすことは驚くほど簡単になりました。ChatGPTに聞けば、アイデアも例え話も情報も、もっともっとと言えば言うだけ増えていきます。基本的に生成AIは「加える」ことが得意です。しかし、その逆の「何を引くか」「どこまで減らせるか」は、AIが苦手な作業です。

苦手というよりも、指示がないと何を残すべきかがうまく判断できないというのが正しい表現かなと思います。そのため、最近は生成AI側から次のアクションのために必要な方向性を聞き出すための追加質問が入るケースもよく見られます。

何を削るか（＝何を残すか意識的な選択のプロセス）を意識した上で、こういうプロンプトを使うと便利だよというものを紹介します。

## 🪓 削る技術の3つのポイント

削る技術には3つのポイントがあります。これは私自身が日頃から意識していることです。

### 最初から短く書こうとしない（まずは出す）

削るためには、より多くの選択肢が必要です。最初から選べるものが少ないと、どうしても核となる考えを見つけることが難しくなってしまいます。たくさんの方向性や選択肢を出し、その中から絞り込んでいく方が自分の核となる部分を見つけ出しやすいです。

文章術なんかでもよく言われる、いきなり完成系の文章を書こうと意識すると、多くの人が最初の段階で手が止まってしまうというのと同じです。最初は書きたいこと、頭に浮かんだことを片っ端から書き出します。構造も文体も無視で、とりあえず全部出してしまう。

その後、「自分が何を伝えたいのか」を明確にするために、選ぶことを通して、自分の思考の軸が少しずつ浮かび上がってきます。

雑多な選択肢がないと、そもそも削ることはできません。  
削る技術の第一歩は、まずは思いつくことをすべて出し切ること。そこから削る方が、結果的に質の高いものが残ります。

### 削る基準を決める

何のために削るのか。その基準を決めておくことが大切です。「捨てる基準」がないと、どれも大事に思えてしまい、結局削れません。

結論をまとめたいのか、行動につなげたいのか、比較したいのか。目的によって、残すべき要素は変わります。

- 結論を強調したいときは「結論と理由1つ」だけ残す
    
- 行動につなげたいときは「次のアクション」と「判断材料」だけ残す
    
- 比較したいときは「選択肢の比較に必要な情報」だけを残す
    

削るためには「目的の再確認」が必要です。「削る」とは、ただ量を減らすことではなく、**意味を強調すること**だからです。基準がないまま削ると、ただ短くなっただけで、伝えたいことがぼやけてしまいます。

### AI任せにしない

削る過程では「なぜこれを残すのか」「なぜこれを削るのか」を自分で判断することで、自分の考えが見えてきます。AI任せにすると、機械的な圧縮になり、自分の考えとは言えません。

つまり「伝えたい一点」を自分で選び取る作業は自分でやるしかないんです。これが削る技術の肝です。

## 🧰 削ることに特化した3つのプロンプト

たくさんの選択肢の中から自分で選び取る作業こそが削る技術の肝とはいえ、大量の情報をゼロから人力で処理するのは手間がかかることも事実です。そこで使いたいのが、「削ることに特化したプロンプト」です。

### 1. 結論だけを残すプロンプト

「何が結論なの…？」を解消するためのプロンプト。

```
この文章から「結論」と「理由1つ」だけを残して、3行にまとめて。
前提説明・装飾・例え話はすべて削ってください。
```

### 2. 行動につなげるためのプロンプト

「で、結局何したらいいの…？」を解消するためのプロンプト。

```
読み手がすぐ行動できるように、「次のアクション」と「判断材料」だけを抽出して2〜4行に圧縮して。
背景や歴史、一般論は削除してください。
```

### 3. 比較するためのプロンプト

意思決定のために「選択肢の違い」だけを残したいときに使うプロンプト。

```
以下の文章から「選択肢の比較に必要な情報」だけを抽出し、
判断に不要な背景説明や主観をすべて削除して、
A/Bの違いと判断軸が一目で分かる形に再構成してください。
```

### 🧪 実際に3つのプロンプトを同じ文章に適用してみる

たとえば、以下のような会議の議事録ドキュメントがあったとします。この内容を、結論だけにする、行動につなげる、比較する、3つの目的別に処理してみます。

```
私たちのチームは、現在のタスク管理の方法を見直す必要があります。

理由は、スプレッドシートとチャットを併用しているために、情報が分散しやすく、タスクの抜け漏れが頻発しているからです。特にリモートメンバーが増えたことで、リアルタイムの状況共有が難しくなり、認識ズレによって再作業が増えています。

さらに、他社ではプロジェクト管理ツールの導入が進んでおり、効率面でも採用面でも遅れをとる可能性があります。もちろん、新しいツールの導入にはコストや学習時間も必要ですが、長期的には業務スピードと透明性の向上につながると考えています。

以上を踏まえて、私たちが今後どの方向に進むべきか検討する必要があります。
```

### 結論だけを残す

```
タスク管理方法を見直すべきです。  
現在の運用では情報が分散し、タスクの抜け漏れが発生しているためです。  
効率改善のために一度立ち止まる必要があります。
```

私は「3行にまとめて」という指示は、さまざまな場面で使っています。もっと短くしたいときは「1文でまとめて」という指示を出す場合もあります。

### 行動につなげる

```
まず、プロジェクト管理ツールの試験導入を検討してください。  
今の運用では抜け漏れが発生しており、情報共有の遅れが作業効率を下げています。  
トライアルを実施し、チームの負荷と改善度を確認することが次のステップです。
```

「ステップバイステップで」や「1つのタスクが10分以下になるように」などを追加で指示すると、かなり細かくタスク分解されるので行動を起こしやすくなります。

### 比較する

```
A：現状維持（スプレッドシート＋チャット）

- コストゼロ
- ただし情報分散により抜け漏れが発生

B：プロジェクト管理ツール導入

- 導入コストと学習が必要
- 情報の一元化で作業効率が上がる可能性が高い

判断の軸は「抜け漏れの削減をどれだけ優先するか」です。
```

製品やサービスの比較をする時は、「表形式で」と指示すると見やすく出力してくれます。

判断軸はこちらで事前に指示することもできます。

## 🏃‍♀️削る技術が身につくと、行動につながる

生成AI時代、「いかに早く大量に」ではなく、「いかに短く、本質だけを抽出できるか」が、行動力や思考力に直結します。

削ることは、“自分を残す”ことです。「何を残し、何を削るか」を判断する力、つまり**「削る」技術とは、意思決定そのもの**なんです。

現在は、iPhoneやiPadのChatGPTアプリでも音声モードがチャットモードに組み込まれているため、わざわざプロンプトをコピーしたりしなくても音声だけで指示することもできます。

たとえば、「結論だけ短くまとめてー」って話しかければ、こんな感じになります。さらに短くしたい場合も「もっと短くして」って伝えるだけです。

以前からChatGPTなどの生成AIを使うときは、よく音声モードを使っています。私が音声モードをよく使うのは、テキスト入力よりも速いというのもありますが、それよりも音声モードの方が自分の考えをざっくばらんに話しやすいという点が大きいです。

機械相手に話しかけるのは、最初は抵抗がある人も多いかもしれません。ですが、一旦慣れてしまえばかなり自然に自分の考えを吐き出せるようになります。

書くことに苦手意識がある人は、話しちゃえばいいと思います。

### 完璧を目指さず、試行錯誤していく

あくまでも、削る技術の肝は「何を削るか」を自分に問い続けることなので、目的さえ見失わなければ、手段はいろいろあっていいはずです。

生成AIに投げるプロンプトも、最初から完璧なプロンプトを渡すというよりも、いろいろ試してみて調整しながら自分のほしいものに近づけていく方がうまくいきます。

問いを立て、広げ、削っていくと、自分の思考の軸がだんだん見えてきます。  
そして次に必要になるのが、その軸を中心に**どう構造化していくか** という技術です。
