# goryugocom プロジェクト

Knowledge Stack（Substack記事）のポータルサイト

最終更新: 2025-12-29

---

## 目的

**読者が感動・共感して有料会員に入りたくなる**サイトを作る。

- 有料会員への便利なナビゲート
- せっかくの資産（470記事、639アトミックノート）を活かして入会しやすく
- 記事・動画の価値を文章と構造で伝える

---

## プロジェクト構成

```
goryugocom/
├── _ai_workspace/           ← プロジェクト管理（AIドキュメント）
│   ├── SITE_PLAN.md         ← サイト全体計画（最重要）
│   ├── ATOMIC_NOTES.md      ← アトミックノート設計・運用
│   ├── SEMINAR_ARCHIVE_WORKFLOW.md  ← セミナー処理ワークフロー
│   └── dailylog/            ← AI作業ログ
│
└── web/                     ← Astroプロジェクト
    ├── src/
    │   ├── content/
    │   │   ├── ks/          ← 記事（470件）
    │   │   ├── atomic-notes/ ← アトミックノート（639件）
    │   │   └── topic/       ← 用語集（3件）
    │   └── pages/
    │       └── index.astro  ← ポータルサイト
    └── scripts/
        └── syncArticles.js  ← 同期スクリプト
```

## 主要ドキュメント

### プロジェクト管理
- **`SITE_PLAN.md`** - サイト全体計画（最重要・まずこれを読む）
- **`ATOMIC_NOTES.md`** - アトミックノート設計・運用
- **`SEMINAR_ARCHIVE_WORKFLOW.md`** - セミナー処理ワークフロー
- **`dailylog/`** - AI作業ログ

### AI作業ログの記録習慣

AIの作業記録は `_ai_workspace/dailylog/` フォルダ内の `AI_YYYY-MM-DD.md` 形式のファイルに毎日記録されます。記録のタイミング、構造、内容に関する詳細な原則は、`CLAUDE.md`の「## デイリーログの記録習慣（重要）」を参照してください。この記録は、AIの思考プロセス、意思決定の根拠、問題解決の経緯、および学びを追跡し、プロジェクトの再現性と引き継ぎを容易にするために極めて重要です。

## 方針

### 完璧を目指さない

- 未整備・途中状態を前提にする
- prefix未整備でも公開する
- 段階的に改善していく

### Astroの役割

- Substackが一次配信元（記事本文）
- Astro側は「索引・ナビ・構造」を担う

### 運用

- 記事作成: Obsidian → Substack公開
- セミナー処理: AI対話で文字起こし→構造化→アトミックノート抽出
- 既存記事: 段階的に整理（一気にやらない）
