# アトミックノート 設計・運用

最終更新: 2025-12-29

---

## 役割

アトミックノートは、このサイトにおいて**思考の断面を公開する場所**である。

**主な役割**:
- 記事を探しやすく、入りやすくするための補助線
- 「この人の考え方が好きだ」と感じてもらう
- 思考の丁寧さ・誠実さ・継続性を可視化
- 「この場所に参加したい」という感情を生む

**位置づけ**:
- 知識ベースではない
- 完成した答え集でもない
- 「ここまで分解して考え続けている」という姿勢を自然に伝える存在

**記事との関係**:
- 記事・動画が主役
- アトミックノートは補助線

## データ構造

### 保存場所

**ソース**（将来的に）:
- ルートの `atomic-notes/` フォルダ（未実装）

**現在の保存場所**:
- `web/src/content/atomic-notes/`（639個）

### frontmatter

```yaml
---
project: an              # 固定値: "an" (アトミックノート)
title: ノートのタイトル    # 必須: ノートの見出しと同じ
prefix:                  # オプション: 空欄OK、RINK用（将来的に整備）
topic:                   # 必須: 配列形式のトピックタグ
  - タグ1
  - タグ2
  - タグ3
source_article: "ks.240106_記事タイトル"  # オプション: 出典記事
source_timestamp: "05:30"                 # オプション: タイムスタンプ
---
```

**重要**:
- `prefix` は空でもOK（整理待ち）
- `topic` は配列形式で複数指定可能
- `source_article` で元記事と紐付け

## RINKとprefixの位置づけ

### RINKとは

- アトミックノートの並び順を管理する思想
- prefix（例: `dv.1a1a2`）でアルファベティカルに並べて「思考の流れ」を作る

### 運用ルール

- **prefixは任意** - 空でもOK
- **prefix空 = 整理待ち** - すぐに公開できる
- **prefix付与 = 棚に並ぶ** - 私or生成AIが少しずつ整備
- **数字が10を超えないように人間側で担保** - 運用ルール
- **Astro側では文字列としてそのままソート** - 意味解析しない

### topicとの関係

- **topicはprefixとは別の軸**
- **topicはprefixを付ける時の参考パラメーター**
- topic = タグ的な使い方（複数指定可能）
- prefix = 並び順の管理（単一値、階層的）

### 見せ方

- **UIやコピーの前面には出さない**
- **読者に理解させるための仕組みではない**
- **裏方として機能** - 並び順・思考の流れを支える内部構造
- **結果として「気持ちよく並んでいる」状態を作る**

## アトミックノート作成ガイド（AI向け）

### 基本方針

#### 重複チェックの徹底

**なぜ重要か**:
- 似たようなノートが既に存在する可能性がある
- 重複したノートは管理が煩雑になり、価値が薄れる
- 既存ノートとの関連性を確認することで、より良いタイトルが思いつく

**具体的な方法**:

1. **キーワード検索**:
```bash
# タイトルに特定のキーワードが含まれるノートを検索
ls "web/src/content/atomic-notes/" | grep "やる気"
```

2. **本文内容の検索（Grep）**:
```bash
# ノートの本文に特定のキーワードが含まれているか確認
grep -r "やる気" "web/src/content/atomic-notes/"
```

3. **類似タイトルの確認**:
```bash
# 正規表現で類似タイトルを検索
ls "web/src/content/atomic-notes/" | grep -E "(やる気|モチベーション|継続)"
```

#### 重複が見つかった場合の対処

**パターン1: 完全に重複**
- 新しいノートは作らない
- 既存ノートへのリンクを追加

**パターン2: 部分的に重複**
- 視点が異なる場合は別ノートとして作成
- タイトルで違いを明確にする
- 関連ノートとして相互リンク

**パターン3: 既存ノートが薄い場合**
- 既存ノートを拡充する方向で検討
- 新しいノートを作る場合は、既存ノートとの違いを明確に

---

### 作成の流れ

1. **セミナー動画から候補ノートをリストアップ**
2. **各候補について重複チェック**
3. **重複がなければノートを作成**
4. **frontmatter（YAML）を正しく記入**
   - `project: an`
   - `title:` （ファイル名と一致）
   - `prefix:` （空でOK）
   - `topic:` （配列形式、最低1つ）
   - `source_article:` （出典記事）
   - `source_timestamp:` （あれば）
5. **本文を記述**（テンプレート参照）

---

### タイトル付けのコツ

- **具体的に** - 「思考法」ではなく「思考を分解する3つのステップ」
- **検索可能に** - よく使われる言葉を含める
- **一意に** - 他のノートと区別できる
- **短く** - 40文字以内を目安

---

### topicの選び方

**原則**:
- 最低1つ、多くても5つまで
- 一般的な用語を使う（独自用語は避ける）
- 記事の `topic` とも連携を考慮

**例**:
```yaml
topic:
  - Obsidian
  - Zettelkasten
  - 知識管理
```

---

### 注意事項

- **prefix は空でOK** - 整理待ちとして公開可能
- **完璧を目指さない** - 後から編集・統合可能
- **出典を明記** - セミナーや記事からの引用元を記録
- **重複チェックを怠らない** - 既存ノートを活用
